# APP 使用方法

本文说明 MacMima 桌面端的日常使用流程。

## 1. 首次启动

首次打开 APP 时，需要配置：

- 后端地址
- 工作台 Key

后端地址示例：

```text
https://your-domain.example/api
```

工作台 Key 是团队隔离标识。同一个后端可以承载多个工作台，不同工作台 Key
的数据互相隔离。建议使用随机生成的长字符串，并只发给同一团队成员。

如果这个后端和工作台 Key 是第一次使用，首个注册用户会成为管理员。
如果后端已经初始化，普通用户需要使用管理员发放的邀请码注册。

## 2. 注册与登录

注册时需要填写：

- 用户名
- 邮箱
- 主密码
- 邀请码，首个管理员注册可不填

主密码非常重要。它不只是登录密码，也是个人密区加密密钥的来源。
如果忘记主密码，已加密的个人密区数据无法被后端恢复。

登录后 APP 会在本地派生解密密钥，并从后端同步密文数据。

## 3. 个人密区

个人密区适合保存：

- 个人模型平台 API Key
- 个人服务器 SSH 登录信息
- 个人数据库账号
- 测试后台账号
- 只属于自己的开发配置

个人密区内容使用主密码派生密钥加密。后端只保存密文、IV、认证标签和必要元数据。

## 4. 共享密区

共享密区适合保存：

- 团队共用的开发环境配置
- 项目共用数据库账号
- 共享测试账号
- 需要让团队成员统一使用的 API Key
- 项目交接资料

管理员可以在系统设置的用户管理中，为用户开启或关闭共享区访问权限。

当前版本的共享密区使用工作台 Key 派生共享密钥。后续计划升级为每个用户公钥包装共享密钥的方式，
进一步减少工作台 Key 泄露后的影响范围。

## 5. 凭证分类

MacMima 支持常见开发凭证类型：

- 服务器
- 网站账号
- API Key
- 数据库
- 其他

个人密区和共享密区都可以按分类筛选和检索。

## 6. 数据库凭证与表关系

新增数据库凭证时，可以保存：

- 数据库类型
- 主机、端口、数据库名
- 用户名、密码
- 连接字符串
- 表名
- 表说明
- 备注

表名和说明可以一键复制，方便粘贴给 AI 编程工具、数据分析工具或项目文档。

示例：

```text
users：系统用户表，保存登录邮箱、角色、状态
credentials：凭证密文表，保存分类、标题、密文和同步元数据
invite_codes：邀请码表，控制团队成员注册
```

## 7. 本地 API

在个人设置中可以开启本地 API。开启后，APP 会监听：

```text
http://127.0.0.1:<port>/v1/credentials
```

其他本机工具可以使用 Local API Key 推送凭证。详细接口见
[本地 API 接入](./LOCAL_API.md)。

## 8. 管理员功能

管理员可以在系统设置中：

- 查看后端地址和工作台 Key
- 创建邀请码
- 管理用户启用状态
- 设置用户是否拥有共享密区访问权限

## 9. 日常安全建议

- 不要把主密码发给任何人。
- 工作台 Key 只发给同一团队成员。
- 离职或项目交接后及时禁用用户。
- 重要 API Key 建议在供应商后台设置最小权限。
- 定期检查共享密区，删除不再使用的凭证。
