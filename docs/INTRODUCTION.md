# MacMima 产品介绍

MacMima 是一个面向 AI 开发者和小型开发团队的桌面密钥库。它解决的不是
“再做一个密码管理器”，而是把 AI 项目开发过程中高频出现的开发凭证、服务器
连接信息、数据库账号、Markdown 请求文档、表结构说明、开发讨论和团队共享配置整理到一个加密工作台里。

## 为什么需要 MacMima

新晋 AI 开发者通常会同时接触很多平台和工具：

- 大模型平台 API Key
- 向量数据库、业务数据库、缓存服务账号
- 云服务器 SSH 私钥、登录用户、端口和备注
- 应用后台账号、测试账号、Webhook Secret
- 团队协作时需要共享的环境变量和配置说明
- API 请求文档、调试记录、常用代码片段
- 数据库表名、字段含义、用途说明

这些信息如果散落在聊天记录、表格、备忘录和浏览器自动填充里，会带来几个问题：

- 找不到：项目一多，很难快速确认哪个 Key 或账号对应哪个服务。
- 不安全：明文复制粘贴和截图转发容易造成泄露。
- 难协作：新人接手项目时，不知道哪些配置可以共享、哪些只能个人使用。
- 难审计：后端、数据库、表结构和密钥之间缺少稳定关系记录。

MacMima 希望成为 AI 开发者的“开发凭证工作台”。

## 先回答最关键的安全问题

MacMima 不是把明文密码、明文 API Key、明文 SSH 私钥直接保存到后端数据库。

凭证正文会先在桌面端加密，再上传到后端。后端数据库保存的是：

- `encryptedData`：加密后的密文
- `iv`：AES-GCM 随机初始化向量
- `authTag`：AES-GCM 认证标签
- 标题、分类、标签、创建时间等用于列表展示的元数据

后端数据库不应该出现明文数据库密码、明文网站密码、明文 API Key、明文 SSH 私钥。
如果只泄露数据库，攻击者拿到的是密文和元数据，而不是可直接使用的凭证。

MacMima Crypto v2 还支持：

- 个人密区叠加本机增强密钥，本机增强密钥不发送给后端。
- 共享密区使用独立共享密区密钥，而不是把新共享数据绑定到工作台 Key 上。
- 旧数据保持兼容，团队可以逐步编辑并重新保存旧共享凭证完成升级。

需要透明说明的是：标题、分类、标签这类列表元数据为了检索和展示是明文保存的，
所以不要把真正的 Secret 写进标题或标签。

## 适合谁

- 正在学习和搭建 AI 应用的个人开发者
- 同时管理多个模型供应商 API Key 的开发者
- 需要频繁连接服务器、数据库、后台系统的小团队
- 希望自托管密钥库，而不是把全部凭证交给第三方 SaaS 的团队
- 需要把数据库账号和表结构说明放在同一个上下文里的 AI 项目团队

## 核心场景

### 个人密区

个人密区用于保存只有自己使用的凭证，比如个人测试 Key、个人服务器、个人数据库账号。
这些凭证正文由桌面端使用主密码派生密钥加密后上传到后端。

### 共享密区

共享密区用于团队协作。管理员可以在系统设置里控制哪些用户拥有共享区访问权限。
同一后端、同一工作台 Key 下的成员可以看到共享内容，用于开发协作、项目交接和环境配置统一。

Crypto v2 下，共享密区可以使用团队本地配置的共享密区密钥加密。该密钥不应写入后端环境、
聊天记录或代码仓库。

### 数据库关系记录

数据库凭证不仅可以保存账号密码密文，还可以记录表名和对应说明。这样开发者在接入 AI
问答、RAG、数据分析、后台管理系统时，可以快速知道哪些表能用、每张表大概做什么。

### 本地 API 导入

MacMima 桌面端可以开启本地 API。其他本机工具可以把凭证推送到 MacMima，
避免每次手工复制粘贴。

### Markdown 文档

开发者可以保存 Markdown 请求文档、API 说明、SQL、Shell、JSON、YAML 和代码块。
文档正文和凭证一样在本地加密，也可以共享到共享密区。

### 开发讨论

同一工作台成员可以在开发讨论中沟通接口、需求和协作问题，支持图片、`@` 在线成员提醒、
新消息提醒、按日期折叠和分页加载历史。讨论区会做基础敏感词脱敏，但不应该用于发送明文 Secret。
